# Storage and Runtime Data Ownership

This page is the canonical map of where Alpha Pod stores things, what is safe to delete, and what must never be committed or lost.

## Quick Reference

| Folder / File | Category | Safe to delete? |
|---|---|---|
| `data/cache/` | Regenerated cache | Yes — refetched on next run |
| `data/exports/` | Generated exports | Yes — recreated by batch runner |
| `data/valuations/` | Generated exports | Yes — recreated by batch runner |
| `data/dossiers/` | Generated dossiers | Yes — recreated by pipeline |
| `data/alpha_pod.db` | Durable local state | No — contains pipeline log, estimate history, risk snapshots |
| `data/stage1_survivors.csv` | Generated artifact | Yes — recreated by screener |
| `data/regime_model.pkl` | Trained model cache | Yes — retrained on next HMM run |
| `config/config.yaml` | Committed config | Never delete — source of truth |
| `config/universe.csv` | Committed PM input | Never delete — defines research universe |
| `config/story_drivers.yaml` | Committed PM state | Never delete — PM-approved story drivers |
| `config/valuation_overrides.yaml` | Committed PM state | Never delete — PM override decisions |
| `config/agent_recommendations_*.yaml` | Generated PM input | Regenerated by agents; safe to delete |
| `.env` | Local secrets | Never commit; never delete without backing up |
| `frontend/dist/` | Build artifact | Yes — recreated by `npm run build` |
| `frontend/node_modules/` | Dependency cache | Yes — recreated by `npm ci` |
| `site/` | MkDocs build | Yes — recreated by `mkdocs build` |
| `.tmp-tests/` | Test scratch | Yes — recreated by pytest |
| `.pre-commit-cache*/` | Hook cache | Yes — recreated by pre-commit |

## Categories In Detail

### Committed Source

These live in Git and are the source of truth. Never gitignore or delete them.

```
config/config.yaml              — WACC params, LLM routing, paths, FRED, screening defaults
config/universe.csv             — research universe (tickers + metadata)
config/story_drivers.yaml       — PM-approved story drivers (written after agent review)
config/valuation_overrides.yaml — PM-approved valuation overrides
config/settings.py              — Python settings loader (imports config.yaml)
.env.example                    — template for local secrets; committed, no real values
```

### Local Secrets

```
.env    — GEMINI_API_KEY, FRED_API_KEY, etc.
```

**Never commit.** Copy from `.env.example` on a new clone. Keep a backup outside the repo.

### Durable Local State (SQLite)

```
data/alpha_pod.db
```

The primary operational database. Contains:
- `pipeline_log` — run history with status, duration, and error details
- `estimate_history` — analyst estimate snapshots over time (revision signals)
- `macro_series` — FRED macro series cache
- `risk_snapshots` — daily portfolio risk records
- `market_data_cache` — yfinance result cache (TTL 4h)
- `universe` — loaded from `config/universe.csv` on first run

**Do not delete** unless you are intentionally resetting all operational history. The screener, valuation, and dashboard all read from this DB. If it is lost, pipeline_log history and estimate revision history are gone permanently.

Test runs create isolated `data/tracker-test-*.db` files. These are safe to delete; they are also gitignored.

### Regenerated Caches (`data/cache/`)

```
data/cache/edgar/               — SEC EDGAR filing text cache (per ticker)
data/cache/ff_factors.parquet   — Ken French factor data (downloaded once)
data/cache/sec_company_tickers.json — SEC CIK → ticker map
data/cache/seed_universe.json   — screener seed data
data/cache/yfinance_info.json   — yfinance metadata cache
```

All safe to delete. Refetched automatically on next run. Deleting forces a full re-pull from upstream sources — useful when data looks stale.

### Generated Exports (`data/exports/`, `data/valuations/`)

```
data/exports/{TICKER}_Standard.xlsx   — per-ticker valuation workbooks
data/valuations/batch_*.xlsx          — batch valuation output
data/valuations/latest.csv            — Power Query feed (latest batch)
data/valuations/json/                 — machine-readable valuation snapshots
```

Recreated by `python -m src.stage_02_valuation.batch_runner`. Safe to delete; you lose the history of past runs but not the ability to regenerate them.

### Generated Dossiers (`data/dossiers/`)

```
data/dossiers/{TICKER} {Company Name}/   — per-ticker dossier folders
```

Recreated by the pipeline on next dossier run. Safe to delete.

### Trained Model Cache

```
data/regime_model.pkl   — HMM macro regime model (3-state GaussianHMM)
```

Recreated by `src/stage_02_valuation/regime_model.py` when not present. Safe to delete; the model will be retrained from FRED data on next run. Loss has no permanent effect.

### Generated PM Input (`config/agent_recommendations_*.yaml`)

```
config/agent_recommendations_CALM.yaml
config/agent_recommendations_IBM.yaml
...
```

Written by agents after a research run. Inputs for PM review — not authoritative until the PM promotes entries to `story_drivers.yaml`. Safe to delete; agents regenerate on next run.

### Build and Tooling Artifacts

```
frontend/dist/          — Vite production build; recreated by npm run build
frontend/node_modules/  — npm dependencies; recreated by npm ci
site/                   — MkDocs output; recreated by mkdocs build
.tmp-tests/             — pytest scratch; recreated per run
.pre-commit-cache*/     — pre-commit hook environments; recreated by pre-commit
.ruff_cache/            — ruff lint cache
.playwright-*/          — Playwright screenshots and logs
```

All safe to delete. None affect persistent state.

## Common Cleanup Commands

**Clear all caches and regenerated data (keep DB and committed config):**
```bash
rm -rf data/cache/ data/exports/ data/valuations/ data/dossiers/ data/stage1_survivors.csv data/regime_model.pkl
```

**Clear test scratch:**
```bash
rm -rf .tmp-tests/ .pytest_cache/
```

**Clear frontend build:**
```bash
rm -rf frontend/dist/ frontend/node_modules/
```

**Full clean (loses pipeline log and estimate history — destructive):**
```bash
rm -f data/alpha_pod.db
```

## What Must Never Be in Git

The `.gitignore` enforces these; verify with `git status` before committing:

- `.env` and any `.env.*` variants (except `.env.example`)
- `data/alpha_pod.db` and test DB files
- `data/cache/`, `data/exports/`, `data/valuations/`, `data/dossiers/`
- `data/stage1_survivors.csv`, `data/regime_model.pkl`
- `frontend/node_modules/`, `frontend/dist/`
- All `__pycache__/`, `.pytest_cache/`, `.ruff_cache/`
- `.agent/`, `.claude/`, `.codex/`, `.gemini/` (local agent state)

If `git status` shows any of the above as untracked or modified, stop and investigate before staging.
